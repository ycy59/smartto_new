2. FSRS-5 알고리즘 구현
   lib/algorithms/fsrs/
----------------------------------------
[fsrs_state.dart]
- StudyGoalState enum: newCard / learning / review / relearning
- FsrsRating enum: Again(1) / Hard(2) / Good(3) / Easy(4)
- FsrsRating.fromFocusScore(double score)
    0~40%  → Again, 40~60% → Hard, 60~80% → Good, 80~100% → Easy
- FsrsConstants: 가중치(17개), 초기 안정성, 망각 곡선 상수
- UnderstandingLevel별 초기 stability
    hard → 2.1173일, normal → 4.2926일, easy → 5.5786일

[fsrs_engine.dart]
- initFromLevel(understandingLevel)
    신규 목표 생성 시 초기화 → nextDue = 오늘 자정 (즉시 오늘 계획에 노출)
- review(stability, difficulty, repetitions, lastReview, focusScore)
    세션 종료 후 MediaPipe 집중도 점수 기반 FSRS 파라미터 업데이트
- currentRetrievability(): 현재 기억 잔존율 계산

3. 우선순위 계산기
   lib/algorithms/priority_calculator.dart
----------------------------------------
- PrioritizedGoal: {goal, score}
- PriorityCalculator.sort(goals)
  우선순위 기준:
    1. overdue_days = 오늘 - next_due (초과 일수)
    2. deadline_bonus = 시험 모드 && 마감 7일 이내 → (7 - 남은일수) 부스트
  정렬: score 높을수록 우선

4. 도메인 엔티티
   lib/domain/entities/
----------------------------------------
[subject.dart]
- id, name, color (Color → int로 DB 저장)

[study_goal.dart]
- id, subjectId, title
- mode: StudyMode (study / exam)
- understandingLevel: UnderstandingLevel (hard / normal / easy)
- dueDate: DateTime? (시험일 or 학습일, null = 자유 학습)
- FSRS 파라미터: stability, difficulty, retrievability, repetitions,
                 state, lastReview, nextDue, createdAt
- todos: List<TodoItem> (DB join 시 채워짐)
- isDue: 오늘 복습 필요 여부
- daysUntilDeadline: D-day 계산

[todo_item.dart]
- id, goalId, text, isDone, position

[study_session.dart]
- id, goalId, startedAt, endedAt
- focusScore: 0.0~1.0 (MediaPipe 집중도)
- durationMinutes

7. Riverpod Provider
   lib/providers/
----------------------------------------
[database_provider.dart]
- databaseHelperProvider, subjectRepoProvider,
  studyGoalRepoProvider, studySessionRepoProvider, todoRepoProvider

[subject_provider.dart]
- subjectListProvider (AsyncNotifier)
  add(name, color) / delete(id)

[study_goal_provider.dart]
- goalsBySubjectProvider (FamilyAsyncNotifier, subjectId 기준)
  add(subjectId, title, mode, understandingLevel, dueDate, todoTexts)
  applyFocusScore(goal, focusScore) → FSRS 업데이트
  delete(goalId)

[today_plan_provider.dart]
- TodayPlanEntry: {subject, goal, priorityScore}
- todayPlanProvider → getDueToday() + 우선순위 정렬 + Subject 색상 join
- toggleTodoDone(todoId, isDone)
- refresh()

[study_session_provider.dart]
- startSession(goalId) → DB에 세션 시작 기록
- endSession(session, focusScore, subjectId)
    → focusScore * 100 → FsrsRating 변환
    → FSRS 파라미터 업데이트
    → todayPlanProvider 갱신
